# Campusitory PR Roadmap

This roadmap keeps development token-aware without isolating context. Every PR should connect to the platform map, preserve ETL GIS Consulting LLC representation, and show measurable frontend, backend, or platform-quality progress.

## Operating Rule

No PR should be treated as an isolated change. Each PR must state how it advances the current phase and what it unlocks for the next PR.

## Immediate Sequence

1. Platform routes foundation
   - Converts the app from a one-page shell into route-based product surfaces.
   - Adds dashboard, directory, profiles, Campus Room, opportunities, research, and AI routes.
   - Establishes shared module data so future pages do not duplicate content.

2. Domain model foundation
   - Adds typed models for users, institutions, courses, sessions, opportunities, and research items.
   - Keeps backend work lightweight before selecting persistence.

3. Institution directory MVP
   - Adds seed data, filters, institution cards, and detail route planning.
   - Improves frontend discovery and prepares backend search contracts.

4. Academic identity onboarding
   - Adds role-based onboarding UI.
   - Prepares profile and verification schema boundaries.

5. Campus Room prototype
   - Adds lecture dashboard panels for attendance, notes, quiz, assignments, Q&A, and chat.
   - Prepares session and participation models.

6. Opportunity engine MVP
   - Adds scholarships, grants, jobs, fellowships, and postgraduate opportunity cards.
   - Prepares eligibility matching structure.

7. Research library MVP
   - Adds research item listings, metadata cards, and access levels.
   - Prepares document storage and citation boundaries.

8. AI academic boundary
   - Adds AI search shell with clear safety, validation, and usage boundaries.
   - Avoids paid API dependency until local product flows are stable.

## Pull Request Standard

Every PR must include:

- Frontend progress
- Backend progress
- Platform quality progress
- User value
- Next PR unlocked
- Testing status
- Risk and rollback
